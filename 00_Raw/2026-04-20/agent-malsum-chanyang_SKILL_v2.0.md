### [CCM 특화 프롬프트 제작 가이드라인]
1. **장르 (Genre)**: `worship instrumental`, `hymn-inspired`, `devotional piano`, `sacred ambient` 고정. K-Pop·Jazz·EDM 등 세속 장르 **절대 금지**.
2. **템포 (Tempo)**: 50~75 BPM 범위 엄수. 빠른 비트 금지.
3. **악기 (Instruments)**: 피아노·스트링·첼로·바이올린·어쿠스틱 기타 중심. 드럼·신디사이저 리드·일렉 기타 금지.
4. **보컬 (Vocals)**: **`no vocals` 기본값** (Instrumental 모드). 오리지널 창작 가사 CCM 제작 시에만 보컬 허용 (60일차 이후).
5. **가사 (Lyrics)**: 초기 30일은 가사 제작 금지 (Instrumental만). 오리지널 CCM 보컬곡 제작 시에만 대표님 또는 Claude 창작 한국어 가사 사용. **기존 찬양곡 가사 차용 절대 금지** (Section 3 참조).

## Section 4. Core Technical Rules

### Rule 1. Music Generation & Structure (Lyria + Suno 듀얼 엔진)
1. **생성 길이 최적화**: 
   - Google Lyria의 길이 제한을 극복하기 위해 **Suno API(비공식 래퍼 gcui-art 기반 로컬 localhost:3000)** 연동을 기본으로 3~5분 이상의 완전한 곡을 생성합니다.
   - 단일 30초 생성 지시가 있을 경우 반드시 '스티칭'이 아닌 '완전한 곡 생성'을 우선 고려합니다.
2. **콘텐츠 구조**:
   - 롱폼 플레이리스트 제작 시 최소 15~20곡 이상의 유니크한 트랙을 확보하고, **루프 기법**으로 60분 이상의 시청 지속 시간을 보장합니다.
   - **시리즈별 길이 기준**: 새벽기도 60분 / QT 30~45분 / 예배 준비 90~120분 / 힐링 60~120분
3. **믹스 마스터링**:
   - 크로스페이드 2초 적용
   - LUFS -14dB 정규화 (YouTube 표준)
   - 첫 3초 시선 고정 효과 확인 (리텐션 핵심)

### Rule 2. API Integration & Error Handling
1. **API 호출 원칙**:
   - Suno·YouTube·Lyria 등 모든 외부 API 호출은 할당량 사전 체크, 중복 호출 방지용 캐싱 기본 적용
   - 비공식 Suno API(localhost:3000) 인증 토큰(쿠키) 수시 검증, 실패 시 멀티 토큰 시스템으로 즉시 전환
2. **CAPTCHA 대응**:
   - Suno 보안 트리거 발생 시 대표님께 즉시 보고
   - 대표님이 수동으로 웹에서 생성한 음원을 대표님 수동 알림 후 에이전트가 다운로드·수집하는 **Manual Fallback Loop** 자동 가동
3. **에러 핸들링**:
   - API 장애·할당량 초과 시 작업 중단 금지. '선제작 후보고' 체계로 대안 루트(다른 플랫폼 API 또는 로컬 에셋 활용) 찾아 프로세스 완수.

### Rule 3. File & Asset Management
1. **분리 보관**: 코드는 루트 유지, 대용량 미디어(mp3, mp4, png)는 `assets/media/`, 구버전 스크립트는 `utils/archive/`에 격리하여 인덱싱 부하 최소화.
2. **인덱싱 제외**: `.gitignore`에 등록된 미디어 및 아카이브 폴더는 분석 대상에서 제외.

### Rule 4. 🆕 Bible Citation Rules
1. **번역본 고정**: **개역개정** 한국 표준 성경 본문 사용 (퍼블릭 도메인)
2. **출처 표기**: 구절 인용 시 **반드시 출처를 먼저 밝히고** 큰따옴표 사용

## Section 5. Project Structure (기존 유지)

- **`/src/`**: 마스터 실행 코드 및 메인 파이프라인 (`main_pipeline.py`, `assemble_*.py`, `factory_*.py`).
- **`/config/`**: 보안 인증 파일 및 설정 보관소 (`.json`, `.env`, `requirements.txt`).
- **`/utils/`**: 기능별 세부 모듈 분리 폴더.
    - `/core/`: 생성 엔진 및 핵심 알고리즘 로직.
    - `/helpers/`: 운영 지원 헬퍼 및 API 인터페이스.
    - `/tests/`: 유효성 검증 및 실험용 코드.
    - `/archive/`: 구버전 레거시 코드 (인덱싱 제외).
- **`/data/`**: 제작 프로세스 중 발생하는 데이터 저장소.
    - `/music/`:
        - `/suno/`: Suno AI 생성 음원 보관.
        - `/lyria/`: Google Lyria 생성 음원 보관.
    - `/videos/`: 렌더링된 최종 영상 파일.
- **`/logs/`**: 시스템 실행 기록 및 에러 분석 로그.
- **`/assets/`**: 원본 리소스 및 플레이리스트 구성 자산.
    - `/media/`: 고화질 이미지 및 배경 소스 (인덱싱 제외).
    - `/playlists/`: 트랙 리스트 메타데이터 및 설정.
- **`/analysis_reports/`**: 데이터 기반 트렌드 분석 리포트 저장소.

## Section 8. Dual-Engine Music Generation Strategy (CCM 특화)

### 8.1 [Lyria 3 Pro] — 프리미엄 단편 전담 (비중 20%)
- **주용도**: 3분 내외 고퀄리티 단편, Shorts 소스, 영상 인트로 30초 BGM, EP 싱글 컷
- **강점**: 뛰어난 음질, 섬세한 감정 표현, 악기 분리감 우수
- **CCM 활용 예**: 새벽기도 인트로 프리미엄 30초, Shorts 60~90초 하이라이트, 오리지널 CCM 보컬곡(60일차 이후)

### 8.2 [Suno V5] — 대량 생산 메인 엔진 (비중 80%)
- **주용도**: 15~25곡 대량 생성 → 크로스페이드로 60분~2시간 롱폼 재생목록 제작
- **강점**: variation 빠른 생성, BPM·악기 프롬프트 제어력 우수
- **CCM 활용 예**: 새벽기도 60분 / QT 30~45분 / 예배 준비 90~120분 / 힐링 60~120분

### 8.3 [Hybrid Production]
- **롱폼**: Suno 메인 + Lyria 인트로 30초 프리미엄 배치
- **Shorts**: Lyria 단편 1곡 클라이맥스 추출
- **오리지널 보컬곡**: Suno V5 vocal 모드 (60일차 이후 시험작)
