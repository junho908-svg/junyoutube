## 2. 수정된 버그 상세

### 🐛 Bug #001: 어린이 탭 말풍선 텍스트 깨짐

#### 증상

어린이 탭 상단의 로고몽 인사 카드에서 말풍선 안의 텍스트가 **세로로 4줄 깨져서** 표시됨.

```
[정상 표시 기대]
"오늘도 멋진 하루를 보내자!"

[실제 iOS Safari에서 표시]
오늘도
멋진 하
루를 보
내자!
```

#### 영향받는 메시지

이 버그는 어린이 탭 인사말 6개 모두에서 발생:
- "오늘은 어떤 성경 이야기 들어볼까?"
- "스티커 모으는 재미가 쏠쏠해요!"
- "퀴즈로 말씀을 더 잘 기억해요"
- "예수님은 너를 정말 사랑하셔"
- "오늘도 멋진 하루를 보내자!"
- "성경 속에는 보물 같은 이야기가 가득해"

#### 근본 원인 (Root Cause)

**파일**: `App.jsx`
**컴포넌트**: `LogomongSpeech` (1363~1398줄)
**원인 코드**:
```jsx
<div className="flex items-end gap-3">
  <Logomong size={size} animate="bounce" glow />  {/* size=200 */}
  <div className="rounded-2xl px-4 py-3 flex-1">
    <p>{message}</p>
  </div>
</div>
```

**원인 분석**:

1. **공간 부족 (Layout Constraint)**
   - iPhone 화면 너비: ~390px
   - 부모 패딩: -32px (양쪽)
   - 로고몽 크기: -200px
   - gap 간격: -12px
   - **남은 말풍선 공간**: 146px
   - 말풍선 안쪽 패딩: -32px
   - **실제 텍스트 영역**: ~114px

2. **iOS Safari의 한글 처리 차이**
   - Chrome/Edge: 한글을 어절(공백 기준)로 줄바꿈
   - **Safari (iOS)**: 한글 어절 단위 미적용 → 글자 단위 강제 줄바꿈
   - "오늘도 멋진 하루를 보내자!" (12자 + 공백) 한 줄에 못 들어감 → 글자 단위로 깨짐

3. **`flex-1`의 Safari 해석 차이**
   - Chrome: 자식 요소가 부모 너비를 자연스럽게 채움
   - Safari: `min-width: 0` 없으면 컨텐츠 최소 너비 무시

#### 수정 방법

**전략**: 큰 사이즈(120px+) 로고몽일 때는 **자동으로 세로 배치**로 전환

```jsx
function LogomongSpeech({ message, size = 90, color = '#E9A94D' }) {
  const isLarge = size >= 120;

  if (isLarge) {
    // 세로 배치 (모바일 친화적)
    return (
      <div className="flex flex-col items-center gap-3">
        <Logomong size={size} animate="bounce" glow />
        <div className="relative rounded-2xl px-5 py-3 w-full">
          {/* 위쪽 화살표 (말풍선 꼬리) */}
          <div style={{ /* 위쪽 삼각형 */ }} />
          <p
            style={{
              wordBreak: 'keep-all',        // 한글 어절 단위
              overflowWrap: 'break-word',
              whiteSpace: 'normal'
            }}
          >
            {message}
          </p>
        </div>
      </div>
    );
  }

  // 작은 사이즈는 기존 가로 배치 유지
  return (...);
}
```

**핵심 변경 3가지**:
1. ✅ `flex-col` → 세로 배치로 충분한 너비 확보
2. ✅ `wordBreak: 'keep-all'` → 한글을 어절 단위로 줄바꿈
3. ✅ 말풍선 꼬리 위치 좌측 → 위쪽으로 자연스럽게 이동

### 🐛 Bug #002: 예배 완료 화면 로고몽 헤더 겹침

#### 증상

5분/10분/15분 예배 완료 후 표시되는 "예배를 잘 마쳤어요" 축하 화면에서 **180px 크기의 큰 로고몽이 상단 헤더 영역으로 튀어나가** 헤더의 BETA 배지, 🔥 출석 일수, ⭐ 스티커 수가 가려짐.

```
[정상 표시 기대]
┌─────────────────────────────┐
│ [🐑 우리집 예배 BETA 🔥0 ⭐0] │ ← 헤더
│                              │
│       [큰 로고몽]            │ ← 본문 영역
│                              │
│   예배를 잘 마쳤어요          │
└─────────────────────────────┘

[실제 표시]
┌─────────────────────────────┐
│ [🐑 우리집 예 ━━━━━━━━━━━]   │ ← 헤더 일부 가려짐
│         ▼                    │
│      [큰 로고몽이             │ ← 헤더 위까지 침범
│       위로 튀어나감]          │
│                              │
│   예배를 잘 마쳤어요          │
└─────────────────────────────┘
```

#### 근본 원인

**파일**: `App.jsx`
**컴포넌트**: `WorshipComplete` (1576~1615줄)
**원인 코드**:
```jsx
<div className="relative inline-block mb-6">
  <div className="text-7xl anim-pop">🌟</div>
  <div className="absolute -right-24 -bottom-4">
    <Logomong size={180} animate="celebrate" glow />
  </div>
</div>
```

**원인 분석**:

1. **`inline-block`의 함정**
   - 부모 컨테이너의 너비가 자식(별 이모지 80px)에 맞춰 작아짐
   - 컨테이너 폭: 약 80px

2. **`absolute -right-24 -bottom-4`의 위치 계산**
   - `-right-24` = `right: -96px` (Tailwind 기준)
   - 컨테이너 우측 바깥으로 96px 튀어나감
   - 로고몽 자체 크기 180px와 합쳐 화면 우측을 크게 벗어남

3. **iOS Safari의 absolute 처리**
   - Chrome: 부모 컨테이너 기준 정확한 위치
   - Safari: `transform` 없이는 GPU 가속 안 받아 계산이 틀어짐
   - 결과: 의도한 우측 하단이 아닌 우측 상단으로 위치 → 헤더 침범

#### 수정 방법

**전략**: `absolute` 위치 지정 제거하고 자연스러운 flex 흐름으로 변경

```jsx
function WorshipComplete({ elapsed, onContinue }) {
  return (
    <div className="pt-8 text-center anim-fade-up">
      {/* 로고몽 + 별: 안전한 flex column */}
      <div className="flex flex-col items-center mb-6">
        <div className="relative">
          <Logomong size={180} animate="celebrate" glow />
          {/* 별을 로고몽 우측 상단에 작게 띄움 */}
          <div
            className="absolute text-4xl anim-pop"
            style={{
              top: '-8px',
              right: '-8px',
              filter: 'drop-shadow(0 2px 8px rgba(255,200,80,0.6))'
            }}
          >
            🌟
          </div>
        </div>
      </div>
      {/* ... 이하 동일 */}
    </div>
  );
}
```

**핵심 변경 3가지**:
1. ✅ `inline-block` → `flex flex-col items-center`로 변경
2. ✅ 로고몽이 메인, 별 🌟은 작은 장식 (우선순위 변경)
3. ✅ 별을 로고몽 컨테이너 안에 안전하게 배치 (우측 상단 -8px)

## 7. 재발 방지 대책

### 7.1 코드 레벨 (이번 패치에 포함)

#### 전역 한글 줄바꿈 적용
```css
body, p, h1~h6, span, div, button, a, li {
  word-break: keep-all;
  overflow-wrap: break-word;
}
```
→ **앞으로 모든 새 컴포넌트에서 자동 적용**

#### iOS safe-area 자동 처리
```css
body {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
}
```
→ **iPhone 노치/홈 인디케이터 자동 회피**

### 7.2 프로세스 개선 (향후)

#### 1. 멀티 디바이스 테스트 필수화
v0.9.2부터 신기능 추가 시:
- ✅ Chrome (데스크탑)
- ✅ Safari (데스크탑)
- ✅ iPhone Safari (필수!)
- ✅ Android Chrome

#### 2. 한글 텍스트 가이드라인
- 새 텍스트 추가 시 모바일 너비(360px)에서 미리 시뮬레이션
- 긴 한글 문장은 의도적으로 `<br/>` 줄바꿈 검토
- 말풍선·카드 등 좁은 공간엔 짧은 메시지

#### 3. CSS 단위 통일
- `absolute` 위치는 픽셀(`px`)로 명시
- Tailwind의 음수 마진(`-right-24`) 사용 시 결과 시뮬레이션
- 모바일 화면에서 컨테이너 밖으로 나가지 않게 주의

#### 4. 정기 회고
- 패치 1주일 후 같은 환경에서 재테스트
- 베타 테스터 피드백 모니터링 (1주일)
- v0.9.2 패치 시 v0.9.1의 수정이 다른 영향 없는지 확인
