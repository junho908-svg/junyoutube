# 🚀 M1 Pro 로컬 AI 연동 일지: Gemma 4 (MLX) + Antigravity

## 🖥️ 내 작업 환경 (Environment)
- **기기:** Apple MacBook Pro (M1 Pro 칩)
- **메모리:** 16GB RAM / 512GB SSD
- **목표:** 클라우드(Gemini 3.1 Flash)와 로컬(Gemma 4 e4b)을 안티그래비티에서 스위칭하며 사용. (기획/분석은 제미나이, 즉각적인 문장 교정은 로컬 젬마 4로 분담)

---

## ❌ 실패 기록 및 원인 분석 (Troubleshooting)

1. **명령어 인식 실패:** `zsh: command not found: pip`
   - **원인:** macOS 기본 환경에서는 `pip` 대신 `pip3`를 사용해야 함.
2. **모델 지원 불가 에러:** `ValueError: Model type gemma4 not supported.`
   - **원인:** `mlx-lm` 라이브러리 버전이 낮아 최신 모델(Gemma 4) 구조 인식 불가.
3. **라이브러리 업데이트 불가 (핵심 원인):** `ERROR: No matching distribution found...`
   - **원인:** 맥북의 **기본 파이썬 버전(3.9)이 구버전**이라 최신 애플 MLX 프레임워크 설치가 차단됨. (MLX는 파이썬 3.9 지원 중단)

---

## ✅ 성공 공식: 최종 해결 워크플로우

구버전 파이썬과의 충돌을 피하기 위해 최신 파이썬(3.11)으로 **'독립된 가상환경(venv)'**을 구축하여 해결.

### Step 1. 최신 파이썬 3.11 설치
```bash
brew install python@3.11
```

### Step 2. 젬마 4 전용 독립 가상환경 생성 및 접속
```bash
# gemma_env 라는 이름의 가상 공간 생성
python3.11 -m venv gemma_env

# 가상 공간으로 입장 (터미널 앞에 (gemma_env) 표시 확인)
source gemma_env/bin/activate
```

### Step 3. 최신 MLX 라이브러리 설치
```bash
pip install mlx-lm
```

### Step 4. 로컬 서버 실행
```bash
python -m mlx_lm.server --model mlx-community/gemma-4-e4b-it-4bit
```
- **성공 로그:** `Starting httpd at 127.0.0.1 on port 8080...`

---

## ⚙️ 안티그래비티(Antigravity) 최종 연동 설정

`antigravity.config.json` 파일을 열고 아래와 같이 수정. 기존 포트(11434)를 MLX 서버 포트(8080)로 변경.

```json
{
    "models": {
        "local": {
            "provider": "openai-compatible",
            "baseUrl": "[http://127.0.0.1:8080/v1](http://127.0.0.1:8080/v1)",
            "model": "mlx-community/gemma-4-e4b-it-4bit",
            "apiKey": "sk-none",
            "temperature": 0.1,
            "maxTokens": 4096,
            "contextWindow": 128000
        }
    },
    "defaultModel": "local",
    "fallback": {
        "enabled": true,
        "model": "gemini 3 flash",
        "trigger": "context_exceeded"
    }
}
```

## 💡 일상 사용 팁 (Daily Usage)

맥북을 껐다 켠 후 젬마 4 로컬 서버를 다시 켤 때는 터미널을 열고 아래 두 줄만 순서대로 입력한다.

1. **방 입장:** `source gemma_env/bin/activate`
2. **서버 실행:** `python -m mlx_lm.server --model mlx-community/gemma-4-e4b-it-4bit`
   
   
   
   # 🍎 MLX란 무엇인가? (Apple MLX Framework)

**MLX**는 애플(Apple)의 머신러닝 연구팀이 직접 개발하여 오픈소스로 공개한 **애플 실리콘(M1, M2, M3, M4 등 M 시리즈 칩) 전용 머신러닝 프레임워크**입니다. 

쉽게 말해, **"맥북이나 맥 스튜디오에서 인공지능 모델을 가장 빠르고 효율적으로 돌릴 수 있게 해주는 애플 공식 도구"**라고 이해하시면 됩니다. 

---

## 1. 탄생 배경: 왜 애플은 MLX를 만들었을까?
기존에 전 세계적으로 가장 많이 쓰이는 인공지능 프레임워크는 '파이토치(PyTorch)'나 '텐서플로(TensorFlow)'입니다. 하지만 이 도구들은 주로 엔비디아(NVIDIA)의 그래픽카드(GPU) 생태계에 맞춰져 있었습니다.

애플의 M 시리즈 칩은 CPU와 GPU가 메모리를 공유하는 매우 독특하고 강력한 구조(통합 메모리)를 가지고 있는데, 기존 도구들은 이 구조의 장점을 100% 끌어내지 못했습니다. 그래서 애플이 **"우리 칩의 성능을 한계까지 끌어다 쓸 수 있는 전용 도구를 직접 만들자"**라고 해서 탄생한 것이 바로 MLX입니다.

## 2. MLX의 핵심적인 특징 (기술적 장점)
- **통합 메모리(Unified Memory)의 완벽한 활용:** 일반적인 컴퓨터는 CPU용 메모리(RAM)와 GPU용 메모리(VRAM)가 나뉘어 있어서, 데이터를 주고받을 때 시간이 오래 걸립니다. 하지만 MLX는 애플 실리콘의 '통합 메모리'를 사용하므로, 데이터를 복사하거나 옮기는 과정(병목 현상) 없이 CPU와 GPU가 즉각적으로 데이터를 처리합니다.
- **친숙한 사용법:** 파이썬(Python)과 C++ 언어를 지원합니다. 특히 파이썬 환경에서는 개발자들이 가장 많이 쓰는 `NumPy`나 `PyTorch`와 코드 작성 방식이 거의 똑같습니다. 기존 개발자들이 새로 사용법을 배울 필요 없이 쉽게 넘어올 수 있도록 설계되었습니다.
- **지연 평가 (Lazy Evaluation):** 연산 명령이 내려졌을 때 바로 계산을 시작하지 않고, 진짜로 그 결과값이 필요해지는 순간에 모아둔 연산을 한 번에 효율적으로 처리합니다. 이 덕분에 메모리 낭비가 줄고 속도가 빨라집니다.

## 3. MLX가 가져온 변화 (사용자 관점)
일반 사용자나 개인 개발자 입장에서 MLX의 등장은 엄청난 변화를 가져왔습니다.
- **로컬 AI의 대중화:** 수천만 원짜리 엔비디아 서버를 빌리지 않아도, 16GB나 32GB 램을 가진 평범한 맥북에서 젬마(Gemma), 라마(Llama) 같은 고성능 언어 모델을 엄청나게 빠른 속도로 돌릴 수 있게 되었습니다.
- **저전력 고효율:** MLX를 사용하면 맥북의 하드웨어 가속을 효율적으로 사용하기 때문에, 무거운 인공지능을 돌려도 발열이나 팬 소음이 적고 배터리 소모가 극적으로 줄어듭니다.

## 💡 요약
**MLX**는 애플이 **"맥(Mac)을 최고의 인공지능 연구 및 구동 기기로 만들겠다"**는 목표를 가지고 내놓은 핵심 무기입니다. 앞서 안티그래비티에 젬마 4를 연결하실 때 `mlx-lm`이라는 라이브러리를 설치하셨던 것도, 바로 이 애플의 강력한 전용 엔진을 사용하기 위한 필수 과정이었습니다.
