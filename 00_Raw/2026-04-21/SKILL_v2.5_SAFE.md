## [MANDATORY] HeyGen 스크립트 작성 규칙

### 규칙 1 – 아라비아 숫자를 한글 음독으로 변환
HeyGen 음성 합성 오류 방지를 위해 **모든** 아라비아 숫자를 한글 음독으로 변환.

| 원문 | 한글 음독 변환 |
|------|---------------|
| 30분 | 삼십 분 |
| 100원 | 백 원 |
| 65세 | 육십오 세 |
| 2주 | 이 주 |
| 3배 | 세 배 |
| 1,500mg | 천오백 밀리그램 |
| 1위 | 일 위 |
| 60대 | 육십 대 |

### 규칙 2 – 한글 수사 예외 목록 (HeyGen·Vrew 공통 보존)
아래 단어들은 **한글 그대로** 사용하며 아라비아 숫자로 바꾸지 않는다.

- **수량/순서**: 첫째, 첫번째, 하나, 둘, 셋, 넷, 한 꼬집, 한 티스푼
- **긴급/중요**: 일일구 (119가 아님)
- **시간/기간**: 네 시간, 다섯 시간 (4시간, 5시간이 아님)

### 규칙 3 – 가독성 및 호흡
- 문장 끝에 반드시 **온점(.)** 사용 → HeyGen이 적절히 쉬게 함
- 강조 부분은 **쉼표(,)**로 호흡 조절 (예: "이것만은, 꼭 기억하세요.")
- 한 문장은 최대 이십오 자 이내 권장 (시니어 가독성)
- 의학 용어 뒤에는 반드시 쉬운 한글 설명 병기

### 규칙 4 – [CRITICAL] 화살표(→) 스크립트 출력 시 절대 사용 금지
HeyGen 음성 합성 오류 유발. **생성되는 스크립트 본문**에서 화살표 대신 다음으로 대체:
- 나열: 쉼표(,) 또는 "그리고"
- 순서: "그다음", "이어서"
- 결과: "그러면", "결국"
- 번호: 원문자(①②③) 또는 "첫째, 둘째, 셋째"

(본 스킬 파일 내부의 지시문 화살표는 Claude용 메타 표기로 무관)
(이미지 마커의 ▶ ◀ 화살표 기호는 음성 합성 대상이 아니므로 예외)

### 규칙 5 – [CRITICAL] 출력 파일 형식
- **허용**: Markdown(.md), 인라인 텍스트
- **금지**: .docx, .doc, .pdf 파일 생성
- 모든 결과물은 대화창 내 Markdown 또는 코드블록으로만 제공

### 규칙 6 – [CRITICAL v2.5 신규] Vrew 자막용 마침표 제거
- HeyGen 스크립트는 온점(.) 유지 (음성 쉼 호흡 필요)
- Vrew 자막용 변환 시에는 **마침표(.) 제거** (시니어 시청자 자막 가독성)
- 쉼표(,)와 줄바꿈은 그대로 유지

### 인트로 훅 3단계 (0:00 ~ 0:30)
1. 공포 자극: 시청자 현재 상황 정확히 지목
2. 타깃 선언: "지금 이 증상이 있다면 당신 이야기입니다"
3. 혜택 약속: "오늘 끝까지 보시면 오늘 밤부터 달라집니다"

### [v2.5 SAFE 신규] 출처 인용 의무 분포

**스크립트 전체에서 ⑭번 출처 인용 패턴을 최소 5회 이상 사용한다.** 권장 분포:
- 본론 1 (0:30~3:00): 1회 (배경 통계)
- 본론 2 (3:00~7:00): 2회 (원인 분석 핵심)
- 본론 3 (7:00~10:00): 1회 (해결책 근거)
- 본론 5 (12:00~14:00): 1회 (보너스 팁 근거)

(더 세부적인 0~15초 하드 후크 설계는 ⑫ 섹션 참조)
(대본 내 이미지 마커 삽입은 ⑬ 섹션 참조)
(의료인 사칭 금지는 ⑰ 섹션 참조)
(AI 환각 차단은 ⑮ 섹션 참조)

## ⑦ 시리즈 연결 설계 (Series Architecture)

> 한 영상이 끝날 때 다음 영상의 씨앗을 심는다.
> 아웃트로 마지막 문장에 자연스럽게 다음 주제 예고를 배치한다.

예시 연결 지도 (대표님 채널 기존 콘텐츠 기반):
- 신장 건강 다음은 혈관 석회화 (인-칼슘 대사 연결)
- 혈관 석회화 다음은 혈압/고혈압 관리
- 고혈압 다음은 뇌졸중 예방
- 야간뇨 다음은 수면의 질
- 수면 다음은 치매 예방 (글림프 시스템)
- 종아리 근육 다음은 낙상 예방
- 침실 폐 건강 다음은 주방 조리흄
- 주방 조리흄 다음은 거실·욕실 공기질

아웃트로 예시:
"오늘 칼슘 이야기를 해드렸는데요. 사실 칼슘 문제의 뿌리는 신장에 있습니다. 다음 영상에서 신장과 혈관의 연결고리, 자세히 알려드릴게요."

## ⑨ 게시 후 성과 진단 프로토콜 (Post-Publish Diagnostic)

> 대표님이 "영상 분석해줘" 또는 YouTube Studio 스크린샷을 공유하시면 즉시 이 프로토콜을 실행한다.
> 목적: 병목 지점을 데이터로 식별하고, 어디를 고쳐야 할지 명확히 알려준다.

### 핵심 지표 5대 축

| 지표 | 무엇을 말하는가 | 시니어 건강 채널 기준값 |
|------|----------------|------------------------|
| **노출수 (Impressions)** | 알고리즘이 얼마나 밀어주는가 | 24시간 내 500+ = 정상 추천 진입 |
| **CTR (노출 클릭률)** | 썸네일·제목이 얼마나 매력적인가 | 4~8% 우수 / 3~4% 보통 / **2% 이하 위험** |
| **AVD (평균 시청 지속)** | 콘텐츠 자체가 얼마나 붙잡는가 | 40%+ 우수 / 30~40% 보통 / **30% 미만 위험** |
| **좋아요 비율 (채널 평균 대비)** | 시청자가 만족했는가 | 채널 평균 이상 = 콘텐츠 품질 양호 |
| **엔딩 화면 CTR** | 후속 유도가 작동하는가 | 0.5%+ 목표 / 0% = 설계 실패 |

### 진단 5단계 플로우

```
1단계: 노출수 확인
  - 500+ 이라면: 알고리즘은 작동 중, 다음 단계로
  - 500 미만이라면: 주제 자체가 카테고리 매칭 실패 또는 태그/제목 SEO 문제

2단계: CTR 확인 (가장 중요한 병목 포인트)
  - 4%+ : 썸네일·제목 정상, AVD로 이동
  - 2~4% : 개선 여지 있음, 다음 영상에 반영 (긴급 아님)
  - 2% 미만 : 즉시 ⑪ 긴급 대응 플레이북 실행

3단계: AVD 확인
  - 40%+ : 콘텐츠 훌륭, 유지
  - 30~40% : 0~30초 드롭 또는 중간 이탈 중 하나, 세부 그래프 확인
  - 30% 미만 : 다음 영상 0~15초 하드 후크 전면 재설계 (⑫ 참조)

4단계: 좋아요 비율 확인
  - 채널 평균 이상 : 콘텐츠 자체는 성공, 문제는 클릭/진입 단계
  - 채널 평균 이하 : 스크립트 내용·톤·정확성 재검토

5단계: 엔딩 CTR 확인
  - 0.5%+ : 정상
  - 0.1~0.5% : 엔딩 화면 위치·문구 개선
  - 0% : 엔딩 설계 실패, ⑫ 섹션의 엔딩 프로토콜 전면 적용
```

## ⑮ [v2.5 SAFE 신규] AI 환각 차단 절차

> **목적**: AI가 만들어낸 가짜 수치·가짜 연구·가짜 통계로 인한 잘못된 의료 정보 차단.
> **YouTube 정책**: 잘못된 의료 정보는 즉시 정책 위반 및 수익화 정지 사유.

### 파트 A — 위험 수치 검증 체크리스트

스크립트에 다음 유형의 수치가 등장하면 **반드시 출처 가능 여부 검증**:

| 위험 수치 유형 | 검증 방법 |
|--------------|----------|
| "삼분의 일", "절반", "X배" 같은 비율 | 실제 학술 자료에서 비슷한 수치 확인 |
| "백팔십 리터", "천오백 밀리그램" 같은 절대 수치 | 출처 가능한지 확인 |
| "칠 년", "이 년" 같은 권장 주기 | 학회 또는 산업 표준 자료 확인 |
| "백 퍼센트", "구십 퍼센트" 같은 효과율 | **사용 금지** (의료 효과 단정형 위험) |
| 특정 질병 발병률 | 질병관리청 또는 KOSIS 데이터 확인 |

### 파트 B — 수치 표현 안전 변환

확인 불가 수치는 다음과 같이 완화:

| 위험 표현 | 안전한 대체 |
|----------|------------|
| "100% 효과가 있습니다" | "효과가 있다는 보고가 있습니다" |
| "절대 안 됩니다" | "권장되지 않습니다" |
| "10명 중 9명이 걸립니다" | "많은 분들에게 나타나는 것으로 알려져 있습니다" |
| "이것만 먹으면 낫습니다" | "도움이 될 수 있다고 보고됩니다" |
| "병이 사라집니다" | "증상이 완화되는 사례가 있습니다" |

### 파트 C — Claude 자체 검증 절차

스크립트 생성 시 Claude는 다음 순서로 자가 검증:

```
1. 모든 구체 수치 식별
2. 각 수치가 일반적 의학 상식 범위 내인지 확인
3. 의심스러운 수치는 "일반적으로 알려진 바로는" 등으로 완화
4. 학술 자료가 명확한 수치만 "[기관명] 자료에 따르면" 형식으로 인용
5. 그 외 수치는 모호한 표현 사용 또는 제거
```

### 파트 D — 환각 의심 사례 (실전)

**의심 신호**:
- 너무 구체적인데 출처 불명 ("정확히 73%가...")
- 알려지지 않은 학술지 인용 ("국제○○○○저널 2023년 5월호에 따르면...")
- 비현실적으로 높은 효과율
- 의료법상 단정 표현 ("X를 먹으면 Y병이 낫는다")

**대응**: 의심 수치는 모두 모호한 표현으로 변환 또는 삭제.

## ⑯ [v2.5 SAFE 신규] 정책 위반 위험 표현 회피

> **목적**: YouTube 잘못된 의료 정보 정책 + 한국 의료법 + 광고주 친화 가이드라인 동시 준수.

### 파트 A — 절대 금지 표현 리스트

스크립트·제목·썸네일·설명란 모두에서 **절대 사용 금지**:

| 금지 카테고리 | 금지 표현 예시 | 대체 표현 |
|-------------|--------------|----------|
| 완치 단정 | "완치됩니다", "낫습니다", "사라집니다" | "도움이 될 수 있습니다", "완화되는 사례가 있습니다" |
| 효과 단정 | "100% 효과", "확실히 효과", "절대 효과" | "효과가 보고됩니다", "효과가 있다고 알려집니다" |

| 진단형 | "당신은 ~병입니다", "이 증상이면 확실히 ~" | "~병의 가능성이 있어 검진을 권합니다" |

| 기적 표현 | "기적의", "마법의", "신기하게도" | "주목할 만한", "보고된 바로는" |
| 개별 의학 조언 | "당신의 경우는 ~" | "일반적으로 알려진 바로는 ~" |

### 파트 B — 안전한 표현 패턴 5종

다음 5가지 패턴을 적극 활용:

```
1. "도움이 될 수 있습니다" (효과를 단정 짓지 않음)
2. "관련 자료에 따르면" (3자 인용으로 책임 분산)
3. "전문가와 상의해보세요" (의료 행위 책임 회피)
4. "보고된 사례가 있습니다" (증거 기반이지만 보편화 회피)
5. "주의해보실 만합니다" (권유형, 강제 아님)
```

   어떤 경우에도 의료인의 진단·처방·치료를 대체하지 않습니다.

### 파트 D — 영상 내 권장 멘트

## ⑳ [v2.4 SAFE 정식 통합] 대본-장면 분리 출력 원칙

> **목적**: HeyGen 입력 효율 극대화 + 편집자 작업 간소화.

### 원칙

- **D 섹션 (HeyGen 대본)**: 순수 대사만 포함 → HeyGen에 즉시 복사 붙여넣기 가능
- **E 섹션 (장면 마커)**: 별도 박스 형태로 분리 → 편집자가 시작 대사로 검색해 타임코드 매칭

### E 섹션 표준 박스 포맷

```
### [장면 N] [분류 ① 시니어 / ② 인포그래픽 / ③ 한국배경] — [섹션명]

- 🎯 **시작 대사**: "..."
- 🏁 **끝 대사**: "..."
- 📍 **프롬프트**: ...
```

### 분리의 장점

1. **HeyGen 효율**: 마커 제거 작업 불필요, 대본 즉시 입력
2. **편집자 효율**: 마커가 박스로 정리되어 한눈에 파악
3. **품질 관리**: 대본 흐름과 장면 구성을 별도로 검토 가능
4. **시각 일관성**: 분류·프롬프트가 일목요연하게 비교됨

## ㉑ [v2.4 SAFE 정식 통합] 독립 영상 검증 원칙

> **목적**: 허위 시리즈 프레임으로 인한 채널 신뢰도 훼손 방지.

### 원칙

풀패키지 제작 시작 **전에 반드시** 다음을 확인:

1. **선행 영상 게시 여부 확인**: 우리 채널에 해당 시리즈 이전 편이 실제로 게시되어 있는가?
2. **레퍼런스와 별개**: 레퍼런스 채널에 시리즈가 있어도 우리 채널에 없으면 독립 영상으로 작성
3. **존재 전제 문구 금지**: 다음 표현은 선행 영상이 있을 때만 사용

| 금지 표현 (선행 영상 없을 때) | 안전한 대체 |
|------------------------------|-----------|
| "지난 영상에서 ~ 알려드렸는데..." | "혹시 이런 생각 해보신 적 있으세요..." |
| "많은 분들께서 놀라셨다고 말씀해 주셨는데..." | (삭제 또는 일반화) |
| "지난 편에서 ~ 다음으로..." | "오늘 알려드릴 ..." |

### 시리즈 연결은 아웃트로에서만

선행 영상 없을 때:
- 본문에는 시리즈 언급 없음
- **아웃트로에만** 후속 영상 예고 자연 심기 (예: "다음 영상에서 ...")

### 게시 후 시리즈 통합 절차

후속 편 게시 시 **이전 편 설명란·고정댓글에 다음 편 링크 추가**:
- 설명란: "🔗 다음 편: [제목] - [URL]"
- 고정 댓글: 동일 형식 추가
